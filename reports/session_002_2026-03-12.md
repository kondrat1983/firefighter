## What Was Done

### Bug Fix — Evidence Card Links
- **Problem:** External link (🔗) on evidence cards in `/alerts/[id]` opened the game store page (`https://store.steampowered.com/app/{ID}/`) instead of the reviews section
- **Fix:** Changed `store_url` in `build_real_alerts()` to `reviews_url = f"https://store.steampowered.com/app/{app_id}/#app_{app_id}_reviews"`
- Updated tooltip on ExternalLink icon: "Open Steam store page" → "View Steam reviews"
- Note: Steam anchor `#app_{ID}_reviews` scrolls to the reviews section on the store page (limitation: no deep-link to individual reviews without author `steamid`)

### GitHub Pages Deployment

**Architecture decision:** GitHub Pages = static only, Python backend can't run there.
Solution: static JSON snapshot of current backend data (`/public/demo-data/`) + client-side fallback.

**Files created/changed:**
- `frontend/next.config.js` — added `output: 'export'`, `basePath: '/firefighter'`, `trailingSlash: true`, `images: { unoptimized: true }`
- `frontend/lib/demoFetch.ts` — fetch utility: tries `http://localhost:8001` with 1.5s timeout, falls back to `/demo-data/*.json`
- `frontend/public/demo-data/` — 22 static JSON files: `alerts.json`, `games.json`, `stats.json`, `alert_100-114.json`, `game_1-4.json`
- `frontend/app/games/[id]/GameDetailClient.tsx` — client component (moved from page.tsx), uses `demoFetch`
- `frontend/app/games/[id]/page.tsx` — thin server wrapper with `generateStaticParams()` for IDs 1–4
- `frontend/app/alerts/[id]/AlertDetailClient.tsx` — client component (moved from page.tsx), uses `demoFetch`
- `frontend/app/alerts/[id]/page.tsx` — thin server wrapper with `generateStaticParams()` for IDs 100–114
- `frontend/app/page.tsx` — updated `FirefighterAPI` to use `demoFetch` instead of direct fetch
- `frontend/components/PasswordGate.tsx` — client-side password gate (`firefighter2026`), uses localStorage
- `frontend/app/layout.tsx` — wrapped `children` in `<PasswordGate>`
- `.github/workflows/deploy.yml` — GitHub Actions: `npm ci` → `npm run build` → upload artifact → deploy to Pages
- `.gitignore` — added `!frontend/lib/` exception (lib/ was ignored by Python packaging rule)

**Build output:** 25 static pages (SSG), all routes pre-rendered.
